# Data Dictionary - Meridian Business Intelligence

Complete field definitions for all three tables. This is the authoritative reference for the synthetic
data universe. No field exists in the CSVs or the
PostgreSQL schema that is not documented here.

---

## Table: clients

One record per client relationship. Primary entity.
All other tables reference this via client_id.

| Field | Type | Valid Values | Business Meaning |
|---|---|---|---|
| client_id | SERIAL | Auto-generated integer | Unique client identifier. Primary key. Referenced in portfolios and transactions. |
| full_name | VARCHAR(100) | Any string | Synthetic full name generated by Faker library. |
| jurisdiction | VARCHAR(50) | China, Singapore, Indonesia, Macau, Brunei | Client home corridor. Drives corridor analytics, FX exposure, and Shariah flag logic. |
| date_of_birth | DATE | YYYY-MM-DD | Client date of birth. Age range 30 to 80. |
| onboarding_date | DATE | YYYY-MM-DD | Date client relationship began. Drives relationship_tenure_months calculation. |
| client_tier | VARCHAR(10) | HNW, VHNW, UHNW | Wealth tier. Drives product access, RM assignment, and co-investment eligibility. |
| investable_aum_usd | NUMERIC(18,2) | HNW: 1M-5M, VHNW: 5M-30M, UHNW: 30M+ | Total investable assets in USD at onboarding. Updated on rebalancing events. |
| primary_currency | VARCHAR(5) | CNY, SGD, IDR, USD, BND | Client base currency aligned to jurisdiction. Drives FX conversion and exposure analysis. |
| product_mandate_type | VARCHAR(20) | Discretionary, Advisory, Execution-Only | Portfolio management model. Discretionary means bank manages without client approval per trade. |
| risk_appetite | VARCHAR(15) | Conservative, Moderate, Aggressive | Client risk tolerance. Drives portfolio construction logic and benchmark selection. |
| shariah_compliant_flag | BOOLEAN | TRUE, FALSE | Activates Islamic finance sleeve. Applies to eligible clients in Indonesia and Brunei. Consistent across all three repos. |
| relationship_tenure_months | INT | 0 and above | Months since onboarding. Feeds CLV calculation and churn risk modeling. |
| assigned_rm | VARCHAR(100) | RM_001 to RM_015 | Relationship manager responsible for client. Feeds RM productivity metrics. |
| co_investment_eligible_flag | BOOLEAN | TRUE, FALSE | Activated for VHNW and UHNW tiers. Triggers access to Meridian Ventures deal flow. |
| last_rebalancing_date | DATE | YYYY-MM-DD | Date of most recent portfolio rebalancing event. |
| annual_income_usd | NUMERIC(18,2) | Greater than 0 | Gross annual income in USD. Used as denominator in DTI calculation. |
| total_debt_usd | NUMERIC(18,2) | 0 and above | Total debt obligations in USD. Used in DTI and DTC calculations. |
| is_active | BOOLEAN | TRUE, FALSE | Active client flag. FALSE indicates formal exit from relationship. |
| exit_date | DATE | YYYY-MM-DD or NULL | Date of formal relationship exit. NULL if client is active. |

## Table: portfolios

One record per client portfolio. References clients via client_id.
In this synthetic universe each client holds one portfolio.

| Field | Type | Valid Values | Business Meaning |
|---|---|---|---|
| portfolio_id | SERIAL | Auto-generated integer | Unique portfolio identifier. Primary key. Referenced in transactions. |
| client_id | INT | Foreign key to clients | Links portfolio to client. One-to-one in this universe. |
| portfolio_name | VARCHAR(100) | MPB-XXXXX format | Synthetic portfolio name. Format: MPB- followed by zero-padded client_id. |
| inception_date | DATE | YYYY-MM-DD | Portfolio start date. Matches client onboarding_date in this universe. |
| base_currency | VARCHAR(5) | USD | All portfolios denominated in USD for cross-corridor consistency. |
| mandate_type | VARCHAR(20) | Discretionary, Advisory, Execution-Only | Portfolio-level mandate. Matches client product_mandate_type. |
| aum_usd | NUMERIC(18,2) | Greater than 0 | Current portfolio AUM in USD. Primary revenue driver. |
| equity_pct | NUMERIC(5,2) | 0.00 to 100.00 | Percentage allocated to equities. |
| fixed_income_pct | NUMERIC(5,2) | 0.00 to 100.00 | Percentage allocated to fixed income. |
| alternatives_pct | NUMERIC(5,2) | 0.00 to 100.00 | Percentage allocated to alternatives - PE, hedge funds, REITs. |
| cash_pct | NUMERIC(5,2) | 0.00 to 100.00 | Percentage held in cash and equivalents. |
| structured_pct | NUMERIC(5,2) | 0.00 to 100.00 | Percentage in structured products. |
| private_credit_pct | NUMERIC(5,2) | 0.00 to 100.00 | Percentage in private credit. Distinct from alternatives. |
| benchmark | VARCHAR(50) | MSCI AC Asia Pacific | Reference benchmark for performance measurement. |
| performance_ytd | NUMERIC(8,4) | Decimal | Year-to-date return as a decimal. 0.0623 means 6.23% YTD return. |
| performance_inception | NUMERIC(8,4) | Decimal | Cumulative return since portfolio inception as a decimal. |
| last_valuation_date | DATE | YYYY-MM-DD | Date of most recent official portfolio valuation. |

Note: allocation percentages across all six asset class columns
must sum to 100.00 for each portfolio. Validated in
01_exploratory.sql Section 3.

## Table: transactions

One record per transaction event. References both clients
and portfolios via foreign keys. The most granular table
in the schema - all flow analysis, fee decomposition,
and FX exposure modeling draws from here.

| Field | Type | Valid Values | Business Meaning |
|---|---|---|---|
| transaction_id | SERIAL | Auto-generated integer | Unique transaction identifier. Primary key. |
| client_id | INT | Foreign key to clients | Direct client reference for corridor-level aggregations. |
| portfolio_id | INT | Foreign key to portfolios | Portfolio reference for portfolio-level analysis. |
| transaction_date | DATE | YYYY-MM-DD | Date transaction occurred. 24-month range. |
| transaction_type | VARCHAR(30) | Inflow, Outflow, Fee, Dividend, Interest, Rebalance, FX-Conversion | Nature of the transaction. Drives all flow analysis. |
| amount_usd | NUMERIC(18,2) | Positive or negative | Transaction amount in USD. Positive is credit. Negative is debit. Outflows and fees are negative. |
| original_currency | VARCHAR(5) | CNY, SGD, IDR, USD, BND | Currency of the transaction before USD conversion. |
| fx_rate_applied | NUMERIC(12,6) | Greater than 0 | Exchange rate used to convert original currency to USD at transaction date. |
| asset_class | VARCHAR(30) | Equities, Fixed Income, Alternatives, Cash, Structured Products, Private Credit | Asset class affected by the transaction. |
| fee_amount_usd | NUMERIC(18,2) | 0 or positive | Fee charged on this transaction in USD. Zero for non-fee transactions. |
| fee_type | VARCHAR(30) | Management Fee, Performance Fee, Advisory Fee, Transaction Fee, NULL | Type of fee. NULL for all non-fee transaction types. |
| rm_id | VARCHAR(100) | RM_001 to RM_015 | Relationship manager who processed the transaction. |
| notes | TEXT | Any string or NULL | Optional transaction notes. NULL for most records. |

## Transaction Type Business Logic

| Type | amount_usd Sign | fee_amount_usd | Typical Frequency |
|---|---|---|---|
| Inflow | Positive | 0 | 30% probability per client per month |
| Outflow | Negative | 0 | 15% probability per client per month |
| Fee | Negative | Positive | Every client every month |
| Dividend | Positive | 0 | Quarterly |
| Interest | Positive | 0 | Monthly for fixed income and private credit holders |
| Rebalance | Zero net | 0 | Triggered by allocation drift |
| FX-Conversion | Varies | 0 | Quarterly for multi-currency corridors |

## Credit Risk Ratio Definitions

Calculated in credit_risk_scoring.py from clients table fields.
Not stored columns - derived at analysis time.

| Ratio | Formula | Threshold - Low | Threshold - Moderate | Threshold - High | Threshold - Critical |
|---|---|---|---|---|---|
| DTI - Debt to Income | total_debt_usd / annual_income_usd | Below 0.30 | 0.30 to 0.50 | 0.50 to 0.70 | Above 0.70 |
| DTC - Debt to Capital | total_debt_usd / aum_usd | Below 0.20 | 0.20 to 0.50 | 0.50 to 0.70 | Above 0.70 |

Risk tier is assigned as the worse of the two ratios.
A client with DTI of 0.25 and DTC of 0.55 is assigned tier C - High.
