# 01 - Raw Data

Synthetic data universe for Meridian Private Bank.
Generated by `04_advanced_analysis/data_generation.py`
using the immutable client entity schema defined in
the Meridian Strategic Master Guidebook v1.0.

This is a carefully constructed
synthetic universe that mirrors real Asia-Pacific private
banking distributions: AUM ranges calibrated to tier
thresholds, transaction volumes proportional to portfolio
size, DTI and DTC ratios following realistic distributions,
and FX flows reflecting actual corridor currency patterns.

---

## Files

| File | Rows | Description |
|---|---|---|
| clients.csv | 500 to 1,000 | Synthetic client records across five Asia-Pacific corridors |
| portfolios.csv | 1 per client | Portfolio allocations and performance per client |
| transactions.csv | 12,000 to 24,000 | 24 months of monthly transaction history |

|---|---|---|

## Transaction Types

| Type | Description |
|---|---|
| Inflow | Client deposits new assets into portfolio |
| Outflow | Client withdraws assets from portfolio |
| Fee | Management, advisory, performance, or transaction fee charged |
| Dividend | Dividend income credited to portfolio |
| Interest | Interest income from fixed income or private credit holdings |
| Rebalance | Internal portfolio rebalancing - no net AUM change |
| FX-Conversion | Currency conversion between client holdings |

## Generation Rules

AUM values align strictly with tier thresholds. A VHNW client
will never have an AUM below USD 5M or above USD 30M. Transaction
amounts are proportional to portfolio size. An HNW client does
not make UHNW-scale transactions. DTI and DTC ratios across the
client population follow a roughly normal distribution centered
on 0.30 to 0.40, with realistic outliers in the high-risk range.

Do not edit CSVs manually. Re-run `data_generation.py` to
regenerate with consistent distributions. Any schema change
requires a Guidebook version update first.

Full field definitions in `docs/data_dictionary.md`.
